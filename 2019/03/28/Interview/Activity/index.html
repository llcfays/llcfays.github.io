Activity

一个 Android 应用通常由多个 Activity 组成，而所有的 Activity 都归属 Task(栈) 管理。
Task 是多个 Activity 的集合，Activity 按照启动顺序排队存入 Back Stack(返回栈)中。

Activity 生命周期

开头说到，Android 通过栈来管理 Activity ,当一个新的 Activity 被创建时，它会被存入栈顶，
以下是 Activity 的生命周期：

  • onCreate()

    Activity 被创建时必须实现的方法，一般用来进行 Activity 内布局的绑定和初始化，当 onCreate() 方法完成后，一定会调用 OnStart()方法。

  • onStart()

    Activity 即将进入用户可见状态，

  • onResume()

    Activity 进入（恢复）用户可见状态，Activity 进入返回栈栈顶。

  • onPause()

    Activity 失去焦点（在技术上是可见状态）。

  • onStop()

    Activity 不可见，新的 Activity 即将别创建（恢复）并且正在覆盖当前 Activity。

  • onRestart()

    Activity 处于 onStop() 状态时重新启动了。

  • onDestroy()

    Activity 被销毁了。

保存 Activity 状态

当 Activity 暂停或者停止的时候，Activity 的状态会被保留。不过，当系统为了恢复内存销毁了某个 Activity 时，
Activity 对象也会被销毁，Activity 状态也就没法保存。在这种情况下，当我们返回该 Activity 时会被重建。
为了确保有关 Activity 状态的重要信息得到保留，系统会先调用 onSaveInstanceState() 方法。系统会向该方法传递一个 Bundle，我们可以
先将有关信息以键值对的形式保存，当系统重建该 Activity 时提取出来，并且恢复。

结束 Activity

  • Activity.finish()

    结束当前 Activity。

  • finishActivity()

    结束之前启动的另一个 Activity。

Activity 启动模式

为了实现不同的启动需求，系统提供了以下四种启动模式：

  • standard(默认)
    在这种模式下启动的 Activity 可以被多次实例化，即每次启动都会创建一个全新的实例。

  • singleTop

    在这种模式启动的 Activity 如果已经存在与栈顶，那么再启动这个 Activity 时，不会创建新的实例，而是重用位于栈顶的那个实例。并且会调用该实例的 onNewIntent() 方法将 Intent 对象传递到这个实例中。

    如果以 singleTop 模式启动的 Activity 的一个实例已经存在与任务桟中，但是不在桟顶，那么它的行为和 standard 模式相同，也会创建多个实例。

  • singleTask

    在这种模式下，会创建一个新的栈，并且 Activity 始终时在栈最底部的，并且该 Activity 启动的其他 Activity 会和该 Activity 同时存在于这个新任务中。如果系统中已经存在这样的一个 Activity 则会重用这个实例，并且调用他的 onNewIntent() 方法。即，这样的一个 Activity 在系统中只会存在一个实例。

  • singleInstance

    在这种启动模式下，栈内有且只有一个相同的 Activity 实例，也就时说如果该实例已经被启动，那么再启动这个 Activity 时，不会创建新的实例，会重用已存在的任务和实例。并且会调用这个实例的 onNewIntent() 方法
