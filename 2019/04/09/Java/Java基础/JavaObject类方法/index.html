Java Object类方法

介绍

Object 类是 Java 中所有类的基类，在 Java.lang（Java 8 version 52.0） 包下面，一共12个方法：

Object()

构造方法

在 Java 中类定义的时候，对于未定义构造函数的类，默认会有一个无参数的构造函数，作为所有类的基类，Object 类自然要反映出此特性。

Class<?> getClass()

getClass() 是一个 Native 方法，返回的是此 Object 对象的类对象/运行时类对象 Class<?>，Object.class 具有相同效果。

“类对象”：

作为概念层次的类，其本身也具有某些共同的特性，如都具有类名称、由类加载器去加载，都具有包，具有父类，属性和方法等。
于是，Java 中有专门定义了一个类，Class，去描述其他类所具有的这些特性，因此，从此角度去看，类本身也都是属于Class 类的对象，为与经常意义上的对象相区分，在此称之为”类对象”。

int hashCode()

hashCode() 方法返回一个整形数值，表示该对象的哈希码值。

hashCode() 具有如下约定：

  • 在Java应用程序程序执行期间，对于同一对象多次调用hashCode()方法时，其返回的哈希码是相同的，前提是将对象进行equals比较时所用的标尺信息未做修改。在Java应用程序的一次执行到另外一次执行，同一对象的hashCode()返回的哈希码无须保持一致；
  • 如果两个对象相等（依据：调用equals()方法），那么这两个对象调用 hashCode() 返回的哈希码也必须相等；
  • 如果两个对象调用 hasCode() 返回的哈希码相等，这两个对象不一定相等。

boolean equals(Object obj)

“==”与”equals”的区别:

  • “==”比较原生类型,如boolean、int、float，就是比较他们的值是否相等。
  • “==”对比两个对象基于内存引用，如果两个对象的引用完全相同（指向同一个对象）时，“==”操作将返回true，否则返回false。
  • equals() 根据具体的业务逻辑来定义该方法，用于检查两个对象的相等性。拿 String 来说就是字符串是否相等。如果在自定义的对象里面，你就需要重写 equals() 方法用于比较哪两个对象相等。

我们列举 “==”与”equals”的区别 来解释 equals() ，需要注意的是，重写equals()方法必须重写hasCode()方法。

Object clone()

clone 方法也是一个 Native 方法，是由 C/C++ 完成的。这个方法的目的是创建并返回此对象一个副本。用官方的话就是：clone 函数返回的是一个引用，指向的是新的 clone 出来的对象，此对象与原对象分别占用不同的堆空间。

  • clone() 在使用的时候需要实现 Cloneable 接口，如果没有实现 Cloneable 接口，并且子类直接调用Object 类的 clone() 方法，则会抛出 CloneNotSupportedException 异常。
  • Cloneable 接口仅是一个表示接口，接口本身不包含任何方法，用来指示 Object.clone() 可以合法的被子类引用所调用。

String toString()

toString()方法返回该对象的字符串

toString() 是由对象的类型和其哈希码唯一确定，同一类型但不相等的两个对象分别调用toString()方法返回的结果可能相同。

void notify()

notify() 方法是一个 Native 方法。

是用于唤醒一个在此对象监视器上等待的线程的，如果有多个线程在此对象上等待，那么只会随机唤醒其中一个，并且在对实现做出决定是发生。

notify方法只能被作为此对象监视器的所有者的线程来调用。一个线程要想成为对象监视器的所有者，可以使用以下3种方法：

- 执行对象的同步实例方法
- 使用synchronized内置锁
- 对于Class类型的对象，执行同步静态方法

一次只能有一个线程拥有对象的监视器。

notify 只能在拥有对象监视器的所有者线程中调用，否则会抛出 IllegalMonitorStateException 异常。

void notifyAll()

跟 notify 一样，唯一的区别就是会唤醒在此对象监视器上等待的所有线程，而不是一个线程。

同样，如果当前线程不是对象监视器的所有者，那么调用 notifyAll 同样会发生 IllegalMonitorStateException 异常。

void wait(long millis)

wait(long millis) 也是 Native 方法。

它会让当前线程等待直到另外一个线程调用对象的notify或notifyAll方法，或者超过参数设置的timeout超时时间。

跟 notify 和 notifyAll 方法一样，当前线程必须是此对象的监视器所有者，否则还是会发生 IllegalMonitorStateException 异常。

wait 方法会让当前线程将其自身放置在对象的等待集中，并且放弃该对象上的所有同步要求。出于线程调度目的，当前线程是不可用并处于休眠状态，直到发生以下四件事中的任意一件:

  • 其他某个线程调用此对象的notify方法，并且当前线程碰巧被任选为被唤醒的线程;
  • 其他某个线程调用此对象的notifyAll方法;
  • 其他某个线程调用Thread.interrupt方法中断了当前线程;
  • 时间到了参数设置的超时时间。如果timeout参数为0，则不会超时，会一直进行等待。

void wait(long var1, int nanos)

和 wait(long millis) 一样，多了一个 nanos 参数，表示额外时间，也就是额外超时多少毫微秒。

void wait()

和前两个 wait() 相同，只不过没有超时时间。

void finalize()

finalize() 作用于垃圾回收，该方法是实例被垃圾回收器回收的时候触发的操作。

额外

  • hashCode 与 equals
    • 两个对象相等 <=> equals()相等 => hashCode()相等
    • hasCode()不相等 => equals（）不相等 <=> 两个对象不相等。
