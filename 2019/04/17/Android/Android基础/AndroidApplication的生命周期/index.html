Android Application 的生命周期

介绍

Application 是 Android 中代表应用程序的类，是和 Activity、service一样，输入 Android 中的一个系统组件。
当 Android 程序启动时，系统会自动创建一个 Application 对象，且只会创建一个。

生命周期

Application 对象的生命周期是整个程序中最长的，也就是整个 Android App 的生命周期

  • onCreate()

    应用程序被创建的时候执行
    一般我们会重写这个方法，然后在方法里初始化配置、资源和第三方服务等。

  • onLowMemory()

    在低内存的时候被调用
    根据当前内存使用情况进行自身的内存资源的不同程度释放，以避免被系统直接杀掉以及优化应用程序的性能体验。

  • onTrimMemory(int level)

    和 onLowMemory() 类似，告知内存使用情况，返回参数为内存级别。

  • onConfigurationChanged(Configuration newConfig)

    应用程序配置信息被改变。
    配置信息是指 Manifest.xml 文件下的 Activity 标签属性 android:configChanges 的值。

  • onTerminate()

    应用程序结束时调用。

onLowMemory() 和 onTrimMemory(int level)

  • onTrimMemory 是 onLowMemory 在 Android 4.0 之后的替代 API。
  • onLowMemory 对应 onTrimMemory 中的 TRIM_MEMORY_COMPLETE 级别。
  • 如果想要兼容 Android 4.0 以下，使用 onLowMemory；否则的话直接使用 onTrimMemory 就好了。

onTrimMemory 的 TRIM_MEMORY_UI_HIDDEN 级别

  • TRIM_MEMORY_UI_HIDDEN 是指 回收UI资源” 。
  • 在 activity onStop() 方法 “回收 Activity” 之前调用。
